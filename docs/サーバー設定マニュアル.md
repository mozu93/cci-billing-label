# CCI請求管理システム　サーバー設定マニュアル

**対象読者：** システム担当者  
**想定環境：** Windows 10 / 11　社内LAN

---

## はじめに

このマニュアルでは、1台のPCをデータベースサーバーとして設定し、
複数の職員PCからCCI請求管理システムを共有して使えるようにする手順を説明します。

```
【完成イメージ】

  サーバーPC（このマニュアルで設定）
  ├─ PostgreSQL（データベース）が常時起動
  └─ 請求データを一元管理

       ↓ 社内LAN

  職員PC①　職員PC②　職員PC③
  それぞれにアプリをインストールするだけでOK
```

---

## 第1章　サーバーPCの準備

### 1-1　推奨スペック

| 項目 | 最低限 | 推奨 |
|------|--------|------|
| OS | Windows 10 | Windows 10 / 11 |
| メモリ | 2GB | 4GB以上 |
| ストレージ空き容量 | 2GB | 5GB以上 |
| ネットワーク | LAN接続（有線推奨） | 有線LAN |

古いPCでも十分動作します。

### 1-2　サーバーPCのIPアドレスを固定する

職員PCから接続するために、サーバーPCのIPアドレスが変わらないよう固定します。

**手順：**

1. サーバーPCで `Windowsキー + R` を押し、`ncpa.cpl` と入力して Enter
2. 使用中のネットワークアダプター（「イーサネット」など）を右クリック →「プロパティ」
3. 「インターネット プロトコル バージョン 4 (TCP/IPv4)」を選択 →「プロパティ」
4. 「次のIPアドレスを使う」を選択して以下を入力

```
IPアドレス：    192.168.1.200　　※社内ネットワークに合わせて設定
サブネットマスク：255.255.255.0
デフォルトゲートウェイ：192.168.1.1　※ルーターのIPアドレス
優先DNSサーバー：192.168.1.1
```

> **確認方法：** 設定前に `Windowsキー + R` → `cmd` → `ipconfig` を実行すると
> 現在のIPアドレスとゲートウェイが確認できます。

5. 「OK」で閉じる

**設定したIPアドレスをメモしておいてください。職員PC設定時に使います。**

## 第2章　PostgreSQLのインストール

### 2-1　インストーラーのダウンロード

1. ブラウザで以下のURLを開く  
   `https://www.postgresql.org/download/windows/`
2. 「Download the installer」をクリック
3. 最新バージョンの「Windows x86-64」をダウンロード

### 2-2　インストール

1. ダウンロードした `postgresql-XX-windows-x64.exe` を実行
2. 「次へ」を押し続ける（インストール先はデフォルトのままでOK）
3. **「Select Components」画面：** 以下にチェックが入っていることを確認
   - ☑ PostgreSQL Server
   - ☑ pgAdmin 4（管理ツール）
   - ☑ Stack Builder（不要だがそのままでOK）
4. **「Password」画面：** `postgres` ユーザーのパスワードを設定  
   ※ **このパスワードは必ずメモしておいてください**
5. **「Port」画面：** `5432`のままでOK
6. 「次へ」→「Install」でインストール開始（数分かかります）
7. 完了画面で「Stack Builder」のチェックを**外して**「Finish」

### 2-3　自動起動の確認

PostgreSQLはインストール時に自動的にWindowsサービスとして登録されます。

**確認手順：**
1. `Windowsキー + R` → `services.msc` と入力して Enter
2. 「postgresql-x64-XX」というサービスを探す
3. **スタートアップの種類が「自動」になっていることを確認**
4. 「自動」になっていない場合：右クリック →「プロパティ」→「スタートアップの種類」を「自動」に変更

これにより、PCを再起動・電源ONした際に自動でPostgreSQLが起動します。

## 第3章　データベースの作成

### 3-1　pgAdmin 4を起動

1. スタートメニューから「pgAdmin 4」を起動
2. 起動時にマスターパスワードを設定するよう求められる場合は、任意のパスワードを設定
3. 左側のツリーで「Servers」→「PostgreSQL XX」をクリック
4. インストール時に設定したパスワードを入力して「OK」

### 3-2　データベースを作成

1. 左ツリーの「Databases」を右クリック →「Create」→「Database...」
2. 「Database」欄に `cci_billing` と入力
3. 「Save」をクリック

### 3-3　接続ユーザーの確認

デフォルトの `postgres` ユーザーをそのまま使用します。  
セキュリティを強化したい場合は専用ユーザーを作成してください（任意）。

## 第4章　社内の他のPCからの接続を許可する設定

> **セキュリティについて：** この設定はあくまで社内LAN内のPCからの接続を許可するものです。  
> インターネット（外部）からはアクセスできません。

### 4-1　postgresql.conf の編集

1. エクスプローラーで以下のフォルダを開く  
   `C:\Program Files\PostgreSQL\XX\data\`  
   （XXはバージョン番号、例：17）
2. `postgresql.conf` をメモ帳で開く
3. `listen_addresses` を検索（Ctrl+F）
4. 以下の行を見つけてコメント（#）を外し、値を変更する

```
変更前：#listen_addresses = 'localhost'
変更後：listen_addresses = '*'
```

5. 上書き保存

> メモ帳で保存できない場合：ファイルをデスクトップにコピー → 編集 → 元のフォルダに戻す

### 4-2　pg_hba.conf の編集

同じフォルダの `pg_hba.conf` をメモ帳で開き、ファイルの末尾に以下の1行を追加する：

```
host    all             all             192.168.1.0/24          md5
```

> `192.168.1.0/24` は社内ネットワークのアドレス帯です。  
> ルーターのアドレスが `192.168.0.1` の場合は `192.168.0.0/24` に変更してください。

保存して閉じる。

### 4-3　PostgreSQLを再起動

1. `Windowsキー + R` → `services.msc` を開く
2. 「postgresql-x64-XX」を右クリック →「再起動」

## 第5章　Windowsファイアウォールの設定

社内の他のPCからアクセスできるよう、ポート5432を開放します。

### 5-1　受信規則の追加

1. スタートメニューで「Windows Defender ファイアウォール」を検索して開く
2. 左側の「詳細設定」をクリック
3. 左ツリーの「受信の規則」を右クリック →「新しい規則」
4. 「ポート」を選択 →「次へ」
5. 「TCP」を選択、「特定のローカルポート」に `5432` と入力 →「次へ」
6. 「接続を許可する」→「次へ」
7. 「ドメイン」「プライベート」にチェック（「パブリック」は外してOK）→「次へ」
8. 名前に `PostgreSQL` と入力 →「完了」

### 5-2　設定の確認（サーバーPC上で実行）

規則が正しく機能しているか確認します。

1. サーバーPCでスタートメニューを右クリック →「Windows PowerShell」を開く
2. 以下のコマンドを入力して Enter

```powershell
Test-NetConnection -ComputerName localhost -Port 5432
```

3. `TcpTestSucceeded : True` と表示されればOK

### 5-3　職員PCから接続確認

続けて、職員PCのいずれか1台でも同様に確認します。

1. 職員PCでPowerShellを開く
2. 以下のコマンドを入力（IPアドレスはサーバーPCの実際のものに変更）

```powershell
Test-NetConnection -ComputerName 192.168.1.200 -Port 5432
```

3. `TcpTestSucceeded : True` と表示されればOK  
   `False` の場合は第5章-1の手順を再確認してください

### 5-4　ウイルス対策ソフトの確認

ウイルス対策ソフト（ウイルスバスター、ESET、Norton 等）を導入している場合、
そちらのファイアウォール機能が別途ポートをブロックしている場合があります。

**確認方法：** 第5章-3の確認で `False` になる場合、ウイルス対策ソフトを
一時的に無効にして再テストする。それで `True` になればウイルス対策ソフト側の設定が原因です。

**対処方法：** 各ウイルス対策ソフトの設定で「ポート5432を許可」するルールを追加してください。
（ソフトにより操作方法が異なるため、製品のマニュアルを参照してください）

## 第6章　Windowsアップデートの設定

業務時間中に突然再起動されないよう設定します。

1. `Windowsキー + I` で設定を開く
2. 「Windows Update」→「詳細オプション」→「アクティブ時間」
3. 業務時間（例：8:00〜18:00）を設定する

これにより、業務時間外にのみアップデートの再起動が行われます。

---

## 第7章　職員PCへのアプリ設定

サーバーの設定が完了したら、各職員PCでアプリを起動して接続設定を行います。

### 7-1　アプリの初回起動

1. CCI請求管理システムを起動する
2. 「初期設定」ダイアログが表示される

### 7-2　接続情報の入力

| 項目 | 入力値 |
|------|--------|
| DB種別 | **PostgreSQL（複数人共有）** を選択 |
| ホスト | サーバーPCのIPアドレス（例：`192.168.1.200`） |
| ポート | `5432` |
| データベース名 | `cci_billing` |
| ユーザー名 | `postgres` |
| パスワード | インストール時に設定したパスワード |

### 7-3　接続テスト

「接続テスト」ボタンをクリックし、「接続に成功しました。」と表示されればOKです。  
「保存して開始」をクリックしてアプリを使い始めます。

## トラブルシューティング

### 接続できない場合のチェックリスト

- [ ] サーバーPCの電源が入っているか
- [ ] サーバーPCとLANで同じネットワークに接続されているか
- [ ] `services.msc` でPostgreSQLサービスが「実行中」になっているか
- [ ] ファイアウォールでポート5432が開放されているか（第5章）
- [ ] `pg_hba.conf` のネットワークアドレス帯が正しいか（第4章）
- [ ] `postgresql.conf` の `listen_addresses = '*'` になっているか（第4章）
- [ ] PostgreSQLを再起動したか（第4章-3）

### サーバーPCを再起動した後につながらない

通常は自動復旧しますが、30秒〜1分ほど待ってから職員PCでアプリを再起動してみてください。
