# 「請求・領収書データ」中心化 設計仕様書

**作成日：** 2026-06-03
**ステータス：** ドラフト（レビュー待ち）
**対象：** `cci-billing`（商工会議所請求書・領収書発行システム）
**位置づけ：** 「まとめて発行」を分かりやすくする UX 改善。2026-06-02「まとめて発行フロー簡略化」の後続。発行ロジック・PDFレイアウト・宛名構造は変えない。

---

## 1. 背景・課題

「まとめて発行」が「名簿登録」から始まるため、**名簿ありき**と誤解される。実際の運用は逆で、

- セミナー参加費のように、**参加者（宛先）が未確定の段階で先に請求内容（箱）を作りたい**。
- その箱に対し、**宛先を随時追加して発行**していく。

また現状は **箱の名前＝業務名**（`Project.name` に業務名をコピー）であり、同じ業務名（例：不動産部会）で複数の箱を作ると一覧で見分けられない。さらに、宛先を随時追加するのに**いつ追加したか**が分からない。

---

## 2. 方針（決定事項：ユーザー合意済み）

1. **用語を「請求・領収書データ」中心に変更**：箱＝請求・領収書データ、その中の宛先一覧＝名簿、と役割を分離。
2. **件名を追加（必須）**：箱ごとに固有の件名を持たせる。`Project.name` を件名に転用（DBスキーマ追加なし）。業務名（`category_id`）は集計キーとして残す。
3. **宛先空でのプレビュー**：箱の請求項目から、宛先空のサンプル請求書／領収書を生成して確認できる（DB保存なし）。
4. **名簿エントリの追加日**：`ProjectMember` に `created_at` を追加し、名簿パネルに「登録日」列を表示。**既定は新しい順**、列クリックで並べ替え可能。

---

## 3. 用語の変更

| 現在 | 変更後 |
|------|--------|
| タブ「名簿登録」 | **請求・領収書データ** |
| ボタン「＋ 新規名簿登録」 | **＋ 新規 請求・領収書データ** |
| ダイアログ「名簿登録／名簿編集」 | **請求・領収書データの登録／編集** |

UI 文言のみの変更。内部のクラス名・テーブル名は変更しない（`ProjectTab`/`projects` 等はそのまま）。

---

## 4. 件名の追加（箱の識別）

### データ
- `Project.name`：これまでの「業務名のコピー」から**自由入力の件名**に意味を変える。スキーマ変更なし。
- `Project.category_id`：**業務名**として継続（レポートの集計キー）。

### 登録／編集ダイアログ（`ProjectFormDialog`）
- 入力項目：**業務名（選択・必須）** ＋ **件名（自由入力・必須）** ＋ 年度 ＋ 備考 ＋ 請求項目テンプレート（1つ以上必須）。
- 保存時 `Project.name = 件名`、`Project.category_id = 業務名`。
- バリデーション：件名が空なら保存不可（「件名を入力してください」）。業務名未選択・テンプレート0件は従来どおり不可。

### 一覧表示（`ProjectTab`）
- 列：**業務名 ｜ 件名 ｜ 状態 ｜ 全件 ｜ 発行済 ｜ 未発行**（現状の「名簿名」を「件名」に、先頭に「業務名」を追加）。
- 業務名は `category` リレーションから取得（無い場合は空表示）。

### レポート類への波及
- `name` が件名になるため、名簿別集計・入金一覧・未払い一覧・再発行の「名簿名」表示は自動的に**件名**になる。
- ヘッダ表記「名簿名」は **「件名」** に変更する（表示文言のみ）。業務名列の追加は本スコープでは行わない（必要になれば別途）。

---

## 5. 宛先空でのプレビュー

箱の中身（請求項目）を、宛先が空の状態でPDF確認できるようにする。

### UI
- 登録／編集ダイアログに **「プレビュー」ボタン** と **書類種別の切替（請求書／領収書）** を追加。
- 押下で、現在ダイアログで選択中のテンプレート群からサンプルPDFを生成して開く。

### 実装
- `pdf_helpers` に**プレビュー専用関数**（例：`generate_preview(lines_data, doc_type, session)`）を追加。
  - 一時的（セッション未追加・コミットなし）の `Issuance` と `IssuanceLine` を組み立てる：
    - 宛先（`recipient_organization`/`recipient_name`）は空。
    - `doc_number` は `"（プレビュー）"`、`amount` は明細合計。
  - 会社・銀行・印鑑は既存 `get_company_and_bank` / `get_default_seal` を使用。
  - 出力は固定のプレビュー用ファイル（毎回上書き）に生成し `open_pdf` で開く。**DBへは一切書き込まない**（`issuance.pdf_path` 更新もしない）。
- 書類種別：請求書＝A4、領収書＝A5（既存レイアウトをそのまま利用）。A4キュー方式は使わず単票で生成する。

## 6. 名簿エントリの追加日

### データ
- `ProjectMember` に `created_at = Column(DateTime, default=datetime.now)` を追加。
- マイグレーション：`connection._migrate` に、`project_members` に `created_at` 列が無ければ `ALTER TABLE project_members ADD COLUMN created_at DATETIME` を追加（既存行は NULL。表示時は空文字で扱う）。

### 取得
- `get_project_members` に並び順オプションを追加（既定：`created_at` の**降順＝新しい順**、NULL は末尾）。既存呼び出し（発行・PDF生成など順序非依存の箇所）への影響はない。

### UI（`ProjectMemberPanel`）
- 名簿テーブルに **「登録日」列** を追加（`YYYY/MM/DD` 表示、NULL は空）。
- **既定は新しい順**で表示。テーブルの**列クリックで並べ替え可能**にする（`setSortingEnabled(True)`）。
  - 日付が文字列ソートで崩れないよう、登録日セルにはソート用の値（ISO文字列または `created_at` を `Qt.ItemDataRole` で保持）を設定する。

---

## 8. スコープ外

- 発行ロジック・入金管理・PDFレイアウト・窓口発行（フリー発行）・年度繰越は変更なし。
- レポートへの業務名列の追加は行わない（件名表示までとする）。
- 既存データの `name`（旧・業務名コピー）の一括書き換えは行わない（開発中・実データ僅少のため。必要なら手直しで対応）。

---

## 9. 受け入れ基準

1. 「まとめて発行」のタブ・ボタン・ダイアログが「請求・領収書データ」基準の名称になっている。
2. 請求・領収書データの登録で**件名が必須**で入力でき、一覧が `業務名 ｜ 件名 …` で表示され、同じ業務名の箱を件名で見分けられる。
3. 登録／編集ダイアログで**宛先空のままプレビュー**でき（請求書／領収書切替可）、DBに発行データが作られない。
4. 名簿パネルに**登録日**が表示され、**既定が新しい順**、列クリックで並べ替えできる。
5. `project_members.created_at` のマイグレーションが既存DBで成功する。
6. 既存テストが緑、本仕様に対応する新テストが通る。
