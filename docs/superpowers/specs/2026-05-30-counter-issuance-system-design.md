# 商工会議所請求書・領収書発行システム 設計仕様書

**作成日：** 2026-05-30  
**ステータス：** 承認済み  
**対象：** 商工会議所の会員向け請求書・領収書発行アプリ（既存請求書アプリとは独立した別アプリ）  
**リポジトリ・ディレクトリ名：** `cci-billing`  
**場所：** `C:\Users\taka\Documents\Gemini\0030Business\cci-billing`

---

## 1. 概要

商工会議所の各部会・検定・事業ごとに、対象会員分の書類（請求書・領収書）を事前にまとめて準備しておき、窓口での手渡し・郵送・メール送付など手段を選んで発行できるようにする。未発行の残数把握・催促状発行にも対応する。

### 技術スタック
- **言語：** Python
- **UI：** PyQt6
- **DB：** PostgreSQL（SQLAlchemy）、サーバーPC1台に集約
- **PDF生成：** ReportLab（既存アプリと同じ）
- **メール：** smtplib（SMTP設定）
- **開発・テスト用DB：** SQLite（SQLAlchemy経由でほぼ同一コード）

## 2. 主要概念（3層構造）

```
カテゴリ
  └─ 請求項目テンプレート（単価・但し書きを登録）
       └─ 事業（特定の発行バッチ・イベント）
            └─ 発行レコード（会員ごとの発行状態）
```

### 2-1. カテゴリ
整理・絞り込み用の分類。例：青年部、女性部、検定、その他

### 2-2. 請求項目テンプレート
単価・税区分・但し書きを1回登録して使い回す。

| フィールド | 内容 |
|-----------|------|
| カテゴリ | 所属カテゴリ |
| 項目名 | 例：青年部会費、珠算検定受験料、コピー代 |
| 単価 | デフォルト単価 |
| 単位 | 式・枚・人・名など |
| 税区分 | 10%・8%・非課税・不課税 |
| 書類種別 | 請求書・領収書・両方 |
| 但し書き | 領収書の但し書き |

数量×単価の計算が必要な項目（コピー代×枚数など）にも対応。

### 2-3. 事業（project）
請求項目テンプレートを1つ以上組み合わせて作る発行バッチ。

| フィールド | 内容 |
|-----------|------|
| 事業名 | 例：2026年度 青年部会費 |
| カテゴリ | 所属カテゴリ |
| 年度 | 例：2026 |
| 事業種別 | **リスト型**（会員名簿あり）／**窓口型**（その場入力） |
| ステータス | draft / active / closed / archived |
| 使用テンプレート | 1つまたは複数（合算発行に対応） |
| 事業ごとの単価上書き | デフォルト単価を事業単位で変更可能 |
| 発行日・期限日 | |

**事業ステータスの意味：**

| ステータス | 意味 | 未納カウント |
|-----------|------|------------|
| draft（下書き） | 年度更新直後・準備中 | カウントしない |
| active（受付中） | 実施中・窓口発行可能 | カウントする |
| closed（終了） | 受付終了・参照のみ | カウントする |
| archived（過去） | 一覧から非表示 | カウントしない |

## 3. 会員マスタ

### 3-1. 会員マスタ (members_master)

| フィールド | 内容 |
|-----------|------|
| 会員番号 | 主キー的役割（変わらない） |
| 事業所名 | 現在の名称 |
| 事業所名フリガナ | |
| 代表者名 | 現在の名称 |
| 代表者名フリガナ | |
| 郵便番号・住所 | |
| 電話・メール | |
| 会員フラグ | 会員 / 非会員 |
| 有効フラグ | |

**宛先ルール：** 事業所名または代表者名のどちらか一方が必須。
- 事業所名のみ → 「○○商事株式会社 御中」
- 代表者名のみ → 「田中 太郎 様」
- 両方 → 「○○商事株式会社 田中 太郎 様」

### 3-2. 名称変更履歴 (member_name_history)
事業所名・代表者名が変更された場合、変更前の名称を記録。発行済み書類のスナップショットには影響なし。

### 3-3. 非会員への発行
会員番号なしで事業所名または個人名だけ入力して発行可能。毎回直接入力するか、非会員として簡易登録するかを選択できる。

### 3-4. 会員の取込方法
- **Excelファイルのインポート**
- **Excelからのコピー貼り付け**（クリップボード貼り付け）
- **手動登録**

## 4. 発行管理

### 4-1. 発行ステータス

```
準備中 → 発行済み → 支払済み
```

| ステータス | 意味 | 対象 |
|-----------|------|------|
| 準備中 | 書類は作成済みだが未発行 | リスト型のみ |
| 発行済み | 書類を渡した・送付済み | 両方 |
| 支払済み | 入金確認済み | 両方 |

**窓口型は「準備中」ステップがなく、発行操作と同時に「発行済み」になる。**

### 4-3. 配付手段
発行時に配付手段を記録する。

| 配付手段 | 内容 |
|---------|------|
| 窓口手渡し | 来所した会員に直接渡す |
| 郵送 | 印刷して郵便で送付 |
| メール送付 | PDFをメールに添付して送信 |
| その他 | 手動記入 |

配付手段は発行記録に保存され、レポートや履歴で確認できる。

### 4-2. 入金消込
「支払済み」への更新時に、入金日・入金額・入金方法（現金・振込など）を記録できる。

### 4-3. 内部統制
- 発行済み書類は変更・削除不可（改ざん防止）
- 全発行記録に担当スタッフ名・発行日時を記録
- 操作ログをDBに保持

## 5. 窓口発行の操作フロー

### フロー①：事業から発行（リスト型）
```
事業「青年部会費2026」を選択
  → 会員一覧表示（未発行のみ / 全員 切替）
  → 名前・会員番号で絞り込み検索
  → 行を選択 → [発行する]
  → 確認ダイアログ → 印刷 / PDF保存
  → ステータス「発行済み」に更新
```

### フロー②：人を起点に横断合算発行
```
[人を検索して発行] ボタン
  → 会員番号・事業所名・事業所名フリガナ・代表者名・代表者名フリガナで検索
  → 全事業横断の未発行一覧を表示（書類種別：請求書 / 領収書 を列表示）
  → 発行する項目にチェック（複数選択＝合算）
     ※ 請求書どうし、または領収書どうしのみ合算可能
     ※ 請求書と領収書を混在させた場合は別々の書類として発行
  → [発行する] → 印刷
  → チェックした全項目のステータスが「発行済み」に更新
```

### フロー③：窓口型（その場入力）
```
事業「珠算検定受験料」を選択
  → 事業所名 または 代表者名を入力（どちらか必須）
  → 数量入力（デフォルト：1）
  → 金額自動計算（単価×数量）
  → [発行する] → 印刷
```

## 7. 年度管理・年度更新

### 年度管理
全事業に `fiscal_year`（例：2026）を持つ。ダッシュボードは年度でフィルタリング可能。過去年度の参照・集計も可能。

### 年度更新機能
「2026年度 → 2027年度」更新時：

- 引き継ぐ事業をチェックで選択
- 会員リストは「そのまま引き継ぐ」か「リセット」かを事業ごとに選択
- 単発事業（視察研修会など）は対象外にできる
- 更新後は全事業が `draft` 状態でコピーされる
- 担当者が「受付開始」ボタンを押したときだけ `active` になる
- **使わない事業は `draft` のまま放置 → 未納にカウントされない**
- 旧年度データは完全保持（削除しない）

## 8. 書類・印刷・PDF

### 対応書類
- 適格請求書（インボイス制度対応・登録番号T番号付き）
- 領収書
- 催促状（未払い会員向け案内文）

### 出力方法
- **その場印刷：** 窓口で1枚ずつ印刷
- **一括PDF生成：** 事業単位で全会員分のPDFを一括生成
- **個別PDF保存：** 発行ごとにPDFファイルを保存

### インボイス制度対応
- 発行元設定にT番号（登録番号）を登録
- 適格請求書の要件を満たすレイアウト（登録番号・税率別内訳・税額）

## 9. メール送付

### 送付方式
- **個別送付：** 一覧から会員を選んで「メール送信」ボタン
- **一括送付：** 「未発行の全会員にメール送信」ボタンで一括送信

### テスト送信機能
- 実際に送付する前に内部メールアドレスへテスト送信できる
- テスト送信時は件名の先頭に「【テスト】」を付与して区別
- テスト送信先メールアドレスは設定画面で登録・変更可能

### メール設定
- SMTPサーバー・ポート・認証情報を設定画面で管理
- 送信元メールアドレス・送信者名を設定
- 件名・本文テンプレートを事業種別ごとに登録・編集可能
- PDFを添付して送信

## 16. 複数職員での共有運用

### 共有方式：サーバーPC1台 + PostgreSQL
事務所内のPC1台をDBサーバーとし、全職員のPCからネットワーク経由でアクセスする。

```
サーバーPC（常時起動）
  └─ PostgreSQL
       └─ counter DB

各職員のPC（最大20台）
  └─ アプリをインストールして起動
  └─ サーバーPCのPostgreSQLに接続
```

### サーバーPCの最低スペック
| 項目 | 要件 |
|------|------|
| CPU | Core i3 相当以上 |
| メモリ | 8GB以上 |
| ストレージ | SSD 256GB以上 |
| OS | Windows 10/11 Pro |
| ネットワーク | 有線LAN（事務所内LAN） |

既存の余剰PCの流用で十分。専用サーバー機は不要。

### 想定利用規模
- 同時起動：最大20人程度
- 同時発行操作：1〜2人（同時多発なし）

### 運用上の注意点
- サーバーPCは使用時間帯は常時起動（スリープ・休止状態をOFF）
- サーバーPCのバックアップは外付けHDDやNASへ定期実行（アプリのバックアップ機能で対応）

### 初期セットアップ
1. サーバーPCにPostgreSQLをインストール（無料）
2. DBとユーザーを作成（手順書を提供）
3. 各職員PCにアプリをインストール
4. 初回起動時にサーバーのIPアドレスと接続情報を設定

### 開発・テスト環境
- 開発側でローカルPostgreSQLを使って開発・テストを完結
- 利用者側に開発環境は不要
- 完成したアプリをインストーラー形式で配布
