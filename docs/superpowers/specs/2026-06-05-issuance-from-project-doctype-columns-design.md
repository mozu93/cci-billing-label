# 登録データから発行：請求書・領収書の2列表示と発行種別の明示

日付: 2026-06-05
対象: まとめて発行 ＞「登録データから発行」タブ（`app/ui/issuance_from_project.py`）

## 課題

1. **発行時にどちらの書類を発行するか分かりにくい。** 上部に「書類種別（請求書／領収書）」コンボはあるが目立たず、発行ボタン（「選択した行を発行」「全員まとめて発行」）に種別が示されない。`_on_project_changed` で名簿テンプレートに応じて自動切替もされるため、今どちらが選ばれているか意識しづらい。
2. **発行後、どちらを発行したか分からない。** 一覧のステータス列は `iss.status`（発行済み/支払済み/未準備）だけで書類種別を示さない。さらに project_member ごとに**最新の発行レコード1件**しか見ないため、請求書を出した後に領収書を出すと、古い方（請求書）の状態が見えなくなる。

運用上、1つの名簿（件名）に対して請求書と領収書の**両方を発行することがある**ため、両種別の状態を同時に把握できる必要がある。

## 決定事項

- 一覧の状態列を「請求書」「領収書」の**2列**に分け、project_member ごとに各種別の最新発行状態を同時表示する（A案）。
- 発行ボタンのラベルを選択中の書類種別に連動させ、何を発行するかを明示する。
- 「未発行のみ」フィルタは**選択中の書類種別**を基準にする。

## 設計

### 一覧の列構成

変更前: `["", 事業所名, 代表者名, ステータス, 発行番号]`（5列）
変更後: `["", 事業所名, 代表者名, 請求書, 領収書]`（5列）

- 「請求書」列・「領収書」列の各セルに、その種別の最新発行レコードの状態を表示する。表記は:
  - 発行レコードなし → `未発行`
  - あり → `<状態> <発行番号>`（例: `発行済 INV-202606-0001`、`支払済 RCP-202606-0001`）
  - 状態の短縮表記: `発行済み`→`発行済`、`支払済み`→`支払済`、`準備中`→`準備中`。
- 列幅: 事業所名（COL_ORG）を Stretch、チェック列（COL_CHK）を固定30px は現状維持。請求書・領収書列は内容が収まる既定幅。

### データ取得（`_load_members`）

project_member ごとに、書類種別別の最新 `Issuance` を取得する:

- `invoice_iss` = その member の `doc_type="invoice"` で最新（`created_at` 降順の先頭）
- `receipt_iss` = その member の `doc_type="receipt"` で最新

各行が保持するデータ（`Qt.ItemDataRole.UserRole`）: `(pm_id, invoice_id, receipt_id)`（無い種別は `None`）。発行処理はチェック行＋選択種別から該当 id を引く。

### 「未発行のみ」フィルタ

- 選択中の書類種別（`_doctype_combo` の currentData）の最新レコードが「発行済み」「支払済み」**でない**行のみ表示する（レコードなし＝未発行も含む）。
- 「すべて」は全行表示。
- 書類種別コンボを切り替えたらフィルタを再適用する（`_doctype_combo.currentIndexChanged` を `_load_members` に接続する）。

### 発行操作の種別明示

- 発行ボタン2つのラベルを選択種別に連動させる:
  - 請求書選択時: 「選択行に請求書を発行」「全員に請求書を発行」
  - 領収書選択時: 「選択行に領収書を発行」「全員に領収書を発行」
- 書類種別コンボ変更時にラベルを更新する小メソッド（例 `_update_issue_button_labels`）を追加し、`_doctype_combo.currentIndexChanged` と初期化時に呼ぶ。

### 発行対象の特定（`_checked_rows` と `_issue_checked`）

- `_checked_rows` は各チェック行から `(pm_id, invoice_id, receipt_id)` を返す。
- `_issue_checked` は選択種別に応じて `issuance_id = invoice_id if doc_type=="invoice" else receipt_id` を使う。`None` なら新規採番（既存ロジックの `create_issuance_for_member(..., doc_type=doc_type)` を流用）→ `mark_as_issued` → `generate_and_open`。
- `_issue_all`（全員まとめて発行）は現状どおり選択種別で `generate_batch_pdf` を呼ぶ。変更なし。

## スコープ外（変更しないもの）

- 発行サービス層（`create_issuance_for_member` / `mark_as_issued` / `generate_batch_pdf`）。
- PDF生成（`generate_and_open`）。
- チェックボックスの Shift 範囲選択（`_CheckableTable`）・ヘッダー全選択。
- 自動種別判定 `_get_doc_type`（名簿テンプレートに領収書があれば初期値を領収書にする挙動）は維持。

## テスト方針

UI ウィジェットのテスト（`qtbot` + `memory_db`）:

- 請求書のみ発行済みの member: 請求書列に `発行済 INV-...`、領収書列に `未発行`。
- 請求書・領収書とも発行済みの member: 両列にそれぞれの番号が出る（古い方が消えない）。
- 「未発行のみ」フィルタ＋書類種別=請求書: 請求書が発行済みの行は出ない。書類種別=領収書に切り替えると、領収書が未発行の同じ行が出る。
- 発行ボタンのラベルが書類種別コンボに連動して変わる。
