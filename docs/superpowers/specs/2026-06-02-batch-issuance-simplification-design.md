# まとめて発行フロー簡略化 設計仕様書

**作成日：** 2026-06-02
**ステータス：** ドラフト（レビュー待ち）
**対象：** `cci-billing`（商工会議所請求書・領収書発行システム）
**位置づけ：** 「まとめて発行」の操作手数を減らす UX 簡略化。事業名簿モデル再設計（2026-05-31）の後続。発行ロジックの本質・DBの宛名構造は変えない。

---

## 1. 背景・課題

現状の「まとめて発行」は手数が多く分かりにくい、とのフィードバック。具体的な不満：

1. 新規事業が `draft` で作られ、**「受付開始（active）」を押さないと発行できない**。
2. 発行が **「準備（採番）」→「発行する」の2段階**。
3. **一括PDF生成が「事業管理」タブにある**（発行操作なのに準備画面にある＝置き場所が不一致）。
4. 一括PDFの**書類種別が暗黙**（最初のテンプレートの種別依存。"両方"だと請求書になり、領収書を一括で出しにくい）。
5. ステータスが draft/active/closed/archived の4段階で、**「終了」「アーカイブ」の意味が分かりにくい**。

---

## 2. 方針（決定事項）

ユーザー合意済み：

- **受付開始を廃止**：事業は作成時から発行可能（`draft` を使わず即 `active`）。
- **発行を1ボタンに統合**：「準備（採番）」と「発行する」を1つの「発行」に。
- **一括発行を「事業から発行」へ移動**：事業管理は準備専用、発行（個別・一括）は「事業から発行」に集約。
- **発行時に書類種別（請求書／領収書）を選択**：個別・一括とも。
- **ステータスを2状態に簡略化（B案）**：`active`（受付中・一覧表示）と `closed`（完了・既定で一覧から隠す）の2つ。`draft`・`archived` は廃止。

---

## 3. ステータスの簡略化

`Project.status` の値を **`active` / `closed`** の2つに集約する。

| 旧 | 新 | 備考 |
|----|----|------|
| draft | （廃止） | 作成時から active |
| active | active | 受付中。既定で一覧表示・発行対象 |
| closed | closed | 完了。既定で一覧から隠す（フィルタで表示可）|
| archived | （廃止→closed に統合） | |

### 変更
- `project_service.create_project`：`status="active"` で作成。
- `project_service`：`activate_project` は不要（削除）。`archive_project` を削除し、`close_project`（→`closed`）と、完了を取り消す `reopen_project`（→`active`）を用意。
- `ProjectTab` の状態フィルタを `["受付中", "完了", "すべて"]`（→ active / closed / None）に変更。既定は「受付中」。
- `ProjectTab` のボタン：`受付開始`・`アーカイブ` を廃止。`完了`（closed 化）と、必要なら `完了を戻す`（active 化）を置く。
- `dashboard.py`：`draft` セクション（「準備中の事業（draft）」一覧と受付開始ボタン）を削除。active 事業の進捗のみ表示。
- `fiscal_year_service.rollover_fiscal_year`：複製する新事業を `status="active"` で作成。`get_rollover_candidates` は `active`/`closed` 対象のままでよい。

---

## 4. 発行の1ボタン化と一括の移動（事業から発行）

`IssuanceFromProjectWidget`（事業から発行）に発行を集約する。

### 4-1. 書類種別の選択
- 画面に **書類種別コンボ（請求書／領収書）** を追加。既定値は事業のテンプレートから推定（領収書テンプレートを含むなら「領収書」、無ければ「請求書」）。
- 個別発行・一括発行とも、このコンボで選んだ種別で発行する。

### 4-2. 個別「発行」1ボタン
- 「準備（採番）」「発行する」を廃し、**「発行」1ボタン**に統合。
- 選択行に対し：未採番なら `create_issuance_for_member(..., doc_type=選択種別)` で採番 → `mark_as_issued(...)` → PDF生成して開く。既に採番済み・未発行なら発行へ。発行済みなら再発行（PDFを開く）。

### 4-3. 「全員まとめて発行」ボタン（旧・一括PDF生成を移設）
- 事業管理にあった「一括PDF生成」を **事業から発行に「全員まとめて発行」ボタン**として移動。
- 選んだ書類種別で、名簿全員ぶんを採番＋発行（status を `発行済み`）＋PDF生成し、保存先を通知。
- `batch_pdf.generate_batch_pdf` を変更：
  - 引数に `doc_type: str` を追加（テンプレート依存の暗黙判定を廃止）。
  - 各名簿行について、未採番なら採番、PDF生成、**`発行済み` に更新**（現状は採番のみで未発行のまま。発行済みに統一）。
  - 宛名が空（事業所名・代表者名とも空）の行はスキップ（現状維持）。

### 4-4. 事業管理から一括PDFを撤去
- `ProjectTab` の「一括PDF生成」ボタンと `_batch_pdf` を削除。事業管理は「準備」（作成・編集・名簿・完了）だけにする。

---

## 5. 変更対象ファイル

| ファイル | 変更 |
|---------|------|
| `app/services/project_service.py` | create_project を active 既定に。activate/archive 廃止、close/reopen 用意 |
| `app/services/fiscal_year_service.py` | 複製事業を active で作成 |
| `app/services/pdf/batch_pdf.py` | doc_type 引数追加、発行済み更新 |
| `app/ui/project_tab.py` | 受付開始/アーカイブ/一括PDF を撤去、完了/完了を戻す・状態フィルタ簡略化 |
| `app/ui/issuance_from_project.py` | 書類種別コンボ追加、発行1ボタン化、全員まとめて発行ボタン追加 |
| `app/ui/dashboard.py` | draft セクション削除 |

DBスキーマ変更なし（status は文字列カラムのまま、使う値が減るだけ）。

---

## 6. スコープ外
- 窓口発行（フリー発行・随時受取）は変更なし。
- 入金管理・レポート・PDFレイアウト・名簿モデルは変更なし。
- 既存DBの status 値の移行：開発は作り直し済み・実データなしのため不要。

---

## 7. 受け入れ基準

1. 新規事業が作成直後から「事業から発行」に表示され、受付開始操作が不要。
2. 「事業から発行」で書類種別（請求書／領収書）を選べる。
3. 「事業から発行」の **1ボタンで採番＋発行＋PDF** が完了する。
4. 「事業から発行」に「全員まとめて発行」があり、選んだ種別で名簿全員ぶんを発行＋PDF生成し、status が発行済みになる。
5. 事業管理に「一括PDF生成」「受付開始」「アーカイブ」が無い。
6. 事業のステータスが `active`/`closed` の2つで、一覧は既定で受付中のみ表示、完了は隠れる（フィルタで表示可）。
7. ダッシュボードに draft セクションが無い。
8. 既存テストが緑、簡略化に対応する新テストが通る。
