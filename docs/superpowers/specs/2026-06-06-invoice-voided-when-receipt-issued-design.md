# 登録データから発行：領収書発行済みなら請求書を「無効」表示にする

日付: 2026-06-06
対象: まとめて発行 ＞「登録データから発行」タブ（`app/ui/issuance_from_project.py`、`app/services/pdf/batch_pdf.py`）

## 課題

通常の流れは「請求書 → 入金 → 領収書」だが、請求書を出さずに領収書だけ発行する事業所もある。その場合、一覧の請求書列が「未発行」のまま残り、「まだ請求書を出していない（要対応）」と誤解する。請求書を出さずに領収書を発行した事業所は「請求書は不要（対応不要）」として区別したい。

## 決定事項

- 請求書が**未発行（レコードなし）**かつ領収書が**発行済み**の事業所について、請求書を「無効」として扱う。
- 「無効」はデータとして記録しない（status="無効" のレコードは作らない）。表示・発行制御の都度、上記条件で判定する。
- 自動無効のみ。取り消しUIは作らない（領収書を発行すれば自動で無効になる）。
- 影響範囲は「登録データから発行」タブの表示と発行制御のみ。他画面（再発行・レポート・入金管理）は変更しない。

## 設計

### 判定（共通）

ある project_member について、最新の請求書 `inv`（doc_type="invoice"）と最新の領収書 `rcp`（doc_type="receipt"）を見て:

```
voided = (inv is None) and (rcp is not None)
```

`_load_members` は既に member ごとに `inv` / `rcp` を取得し、行データ `(pm_id, invoice_id, receipt_id)` を保持している。`voided` はこの値から導ける（`invoice_id is None and receipt_id is not None`）。

### 変更点（すべて「登録データから発行」内）

1. **請求書列の表示**（`_load_members`）
   - 請求書セルのテキストを次にする: `voided` なら `"無効"`、そうでなければ従来どおり `_cell_text(inv)`（未発行／状態+番号）。
   - 領収書セルは従来どおり `_cell_text(rcp)`。

2. **「未発行のみ」フィルタ（書類種別=請求書）**（`_load_members`）
   - 現状は選択種別の最新が「発行済み」「支払済み」のとき非表示。これに加え、書類種別が請求書で `voided` の行も非表示にする（対応不要のため）。
   - 書類種別=領収書のときの挙動は変更しない。

3. **個別の請求書発行**（`_issue_checked`、doc_type=="invoice"）
   - チェックされた行が `voided`（`invoice_id is None and receipt_id is not None`）なら、その行は請求書を発行せずスキップする。
   - doc_type=="receipt" のときはスキップ判定しない（従来どおり）。

4. **「全員に請求書を発行」**（`_issue_all` → `generate_batch_pdf`、doc_type=="invoice"）
   - `generate_batch_pdf` で、doc_type=="invoice" のとき、請求書が未発行（invoice レコードなし）かつ領収書が発行済みの member をスキップする。
   - `generate_batch_pdf` は当画面の `_issue_all` からのみ呼ばれる（他に呼び出し元なし）。

### 領収書発行側

領収書の発行操作自体は変更しない。領収書を発行すれば、次回の一覧表示で請求書が自動的に「無効」になる（判定が動的なため）。これが「自動無効・取り消し不要」と整合する。

## スコープ外（変更しないもの）

- 他画面（再発行 `reissue_tab` ・レポート `report_tab` ・入金管理 `payment_dialog`）。
- `Issuance.status` への "無効" 値の追加・無効レコードの生成。
- 取り消しUI（無効を解除して通常の未発行に戻す操作）。
- 領収書側の表示・発行ロジック。

## テスト方針

UIテスト（`qtbot` + `memory_db`）:

- 請求書未発行・領収書発行済みの事業所 → 請求書列が「無効」と表示される。
- 請求書発行済みの事業所 → 従来どおり「発行済 INV-...」（無効にならない）。
- 「未発行のみ」＋書類種別=請求書 → `voided` の事業所は一覧に出ない。書類種別=領収書では従来どおり。
- `_issue_checked`（請求書）で `voided` の行をチェックして発行しても、その member に新しい請求書が作られない。

サービステスト（`db_session`）:

- `generate_batch_pdf(..., doc_type="invoice")` で、請求書未発行・領収書発行済みの member は請求書が作られずスキップされる（請求書ありや請求書未発行・領収書なしの member は従来どおり発行される）。
